## 대용량 텍스트 자원을 활용한 한국어 형태소 임베딩의 모델별 성능 비교 분석

- Comparative Analysis of Various Korean Morpheme Embedding Models using Massive Textual Resources
- 저자: 이다빈, 최성철
- 년도: 2019.3
- 출처 : DB Pia (국내)

### 요약

- 단어 임베딩 모델 비교 : Word2-Vec[1], GloVe[2], FastText[3]
- 영어는 word2vec 이 효과적
  - 영어는 대부분의 형태소 자체를 하나의 단어로 사용할 수 있는 고립어에 해당한다
  - 단어 임베딩 생성시 어절 단위를 기반으로 문장들을 기반으로 한다
- 한국어는 fasttext 가 효과적
  - 어절 단위 띄어쓰기가 아닌 형태소 단위를 기반으로 단어 임베딩을 생성하여야 한다
  - 한국어의 경우 어근과 접사의 결합으로 하나의 어절이 되는 교착어에 해당되기 때문
- FastText
  - 기존의 Word2Vec의 Skip-Gram을 변형하여 만들어진 모델이다
  - 유사점: Word2Vec의 Skip-Gram 알고리즘과 중심이 되는 단어의 앞과 뒤 문맥에 등장하는 단어들을 유추하도록 학습하는 모델이라는 점에서 유사하다
  - 차이점: 단어가 가지는 형태적인 정보 또한 유사도 함수를 바꿈으로써 학습이 기능할 수 있도록 개선한 알고리즘이다

### 의견

- NLP 과정에서 FastText 를 써야하는 이유 제시

### 초록

단어 임베딩은 컴퓨터가 자연어를 인식할 수 있도록 하는 변환 기법으로 기계번역, 개체명 인식 등 기계학습을 바탕으로 하는 자연어 처리 분야에서 다양하게 사용되고 있다. 단어 임베딩을 생성하는 다양한 단어 임베딩 모델들이 존재하지만 이러한 모델들을 동일한 조건에서 성능을 비교 분석한 연구가 미비하다. 본 논문에서는 한국어 형태소 단위 띄어쓰기를 기반으로 하여 활발하게 사용되고 있는 모델인 Word2Vec의 Skip-Gram과 CBOW, GloVe, FastText의 성능을 비교 분석한다. 뉴스 대용량 말뭉치 및 세종 말뭉치를 바탕으로 실험한 결과 FastText가 가장 높은 성능을 확인할 수 있었다.

- 키워드: 단어 임베딩, 자연어 처리, Word2Vec, GloVe, FastText

### 내용

-  내부 평가 방법 중 의미 관련성과 단어 유추 방법을 활용하여 단어 임베딩 모델 별 성능 평가를 진행
- 세종 말뭉치와 뉴스기사 데이터로 평가 진행
- 성능 : WordSim-353 평가 셋과 Google analogy 평가 셋
  - FastText가 300차원, 200차원일 때 각각 62.97, 82.88으로 가장 높은 성능을 보였다 (최소 100차원 이상, 200차원이 최대차)
  - 윈도우 크기에 따른 단어 임베딩 모델 별 성능은 WordSim-353을 평가한 경우 윈도우 크기이 10인 FastText가 62.97으로 가장 좋은 성능을 보였다 (최소 윈도우 크기 5이상, 7이 최대차)
